# BlogPersonalWebsite
# 一、需求规划阶段（定方向，地基）

## 明确核心定位
个人博客。
### 梳理功能 & 页面清单
#### 核心功能
- 照片，视频等媒体数据分类，储存
    区分核心必备功能和锦上添花功能，列出所有页面（首页、列表页、详情页、个人中心、后台等），避免后期盲目加功能。
    竞品分析
    确定制作方式：自学开发。
    预算、开发工期；
    初步规划域名、服务器选型。

# 二、原型设计阶段（搭框架，定交互）

    绘制站点地图
    梳理全站页面层级、跳转关系，理清整体结构。
    制作线框原型
    使用墨刀、Figma、Axure 等工具做低保真原型，确定每个页面的布局、按钮、表单、弹窗、交互逻辑。
    原型评审修改
    核对流程是否通顺、功能是否完整，反复调整直到逻辑无问题。

# 三、技术架构选型（定技术，搭底层环境）
根据网站复杂度选择技术栈，新手 / 简易站可大幅简化：

    PC + 手机
    前端：Vue框架，
    后端：Java+Python
    数据库：MySQL
    部署载体：个人服务器，静态托管平台：GitHub、Gitee
    第三方服务：提前规划短信、登录、支付、统计、地图等外接接口。

# 四、UI 视觉设计（做样式，出效果图）

    确立视觉规范
    定义网站主色调、字体、图标、版式、整体风格（商务 / 简约 / 潮流等）。
    输出高保真设计稿
    完成全站 PC 端、移动端页面设计，包含弹窗、按钮、状态样式。
    素材整理
    导出图片、图标、切图等资源，交付开发环节。

# 五、代码开发阶段（写代码，实现功能）

    前端开发：还原设计稿，编写页面样式、交互、动画，做好多浏览器、多机型兼容。
    后端开发：搭建服务、编写接口、对接数据库，实现登录、发布、支付、权限等业务逻辑。
    前后端联调：打通数据交互，确保页面和后台数据正常同步。
    管理后台开发：实现内容发布、用户管理、数据查看、权限控制等运维功能。

# 六、全面测试阶段（查漏洞，修 BUG）
这是上线前关键一步，缺一不可：

    功能测试：逐个点击链接、按钮、表单，验证所有功能正常可用。
    兼容性测试：测试不同浏览器、手机系统 / 机型的显示与交互。
    性能测试：检测页面加载速度、高并发下是否卡顿。
    安全测试：排查 SQL 注入、越权访问、爬虫攻击等安全隐患。
    内测试用：小范围邀请他人体验，收集问题并批量修复 BUG。

# 七、上线前置准备（国内核心：备案）
中国大陆使用国内服务器，此步骤强制要求：

    域名注册 + 实名认证：选购域名，在服务商完成实名。
    网站备案：提交主体资料至通信管理局审核，周期一般7~20 天，备案失败无法对外访问。
    服务器配置：搭建运行环境、设置防火墙、安全组。
    域名解析：将域名绑定到服务器 IP。
    配置 HTTPS：申请 SSL 证书，开启全站加密（现在主流网站标配）。
    初始化数据：填充测试内容、配置基础信息。

# 八、正式部署上线（对外开放访问）

    上传代码与资源到服务器，完成项目部署。
    全站最终巡检：复查所有页面、链接、功能、样式。
    开放公网访问，网站正式对外上线。
    部署监控工具：开启访问统计、错误日志监控。

# 九、运维与迭代阶段（长期运营，持续优化）

    日常维护：定期备份数据库与代码（重中之重）、更新内容、处理服务器故障。
    安全防护：定期修复系统漏洞、抵御攻击、防止网站被挂马。
    数据分析：查看访问流量、用户行为，找出体验短板。
    版本迭代：根据用户反馈、业务需求，新增功能、改版页面、优化性能。

# 不同类型网站流程简化参考

    纯静态网站（个人展示、简单博客）
    可省略后端、数据库、复杂原型 → 需求→设计→静态页面开发→域名 + 托管→直接上线，流程最短。
    复杂动态网站（商城、社区、管理系统）
    每一步都需细化，测试、安全、备案、数据备份必须重点投入。
